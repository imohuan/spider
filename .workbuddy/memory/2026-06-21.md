## WebSocket 500 + parsers API 数据丢失修复（2026-06-21 16:03）
- 问题：每次 WebSocket 连接断开时 werkzeug 报 `AssertionError: write() before start_response` (500)，之后 `/api/parsers` 返回空
- 根因 2 层：
  1. **WebSocket AssertionError**: werkzeug 3.x `execute()` 在 WSGI app 返回后若 `start_response` 从未被调用，写入空响应触发断言。`SimpleWebSocketWSGI` 绕过 WSGI 层在 raw socket 上完成 WebSocket 握手，从不调用 `start_response`
  2. **Parsers 数据丢失**: werkzeug ThreadingWSGIServer 复用线程池，WebSocket 线程崩溃后 `_app_ctx_stack`（threading.local）残留可能导致 `current_app` 指向错误实例
- 修复（`web/app.py`）：
  1. 模块级 monkey-patch `SimpleWebSocketWSGI.__call__`：WebSocket 握手前 preemptively 调用 `start_response('101 Switching Protocols', [])` 满足 werkzeug 契约
  2. `web/api/parsers.py` `_get_registry()` 增加防御性诊断日志：缺失时打印 app id + config keys
- 修复（`dev.py`）：补充 `ParserRegistry` 初始化，此前 CRAWLER_COMPONENTS 缺少 `registry` key
- 全量 413 测试通过
