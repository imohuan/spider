## 09:40 修复 Ctrl+C 关闭 dev.py 的 RuntimeError
- 问题：`dev.py` 按 Ctrl+C → `_shutdown` → `stop_image_worker` → `cancel_all_tasks` → `loop.run_until_complete()` 报 `RuntimeError: This event loop is already running`
- 根因：`_img_loop` 在 worker 线程里跑着 `run_until_complete(img_worker.run())`，主线程又调 `loop.run_until_complete()` 试图 cancel tasks — 同一个 loop 不能被两个线程同时 `run_until_complete`
- 修复 `core/bootstrap.py` 两处：
  1. `cancel_all_tasks`：增加 `loop.is_running()` 检测，running 时走 `asyncio.run_coroutine_threadsafe(_drain(), loop)` 在 loop 所在线程内执行 cancel + gather
  2. `stop_image_worker`：调整顺序 — `worker.stop()` → `loop.call_soon_threadsafe(loop.stop)` → `thread.join()` → `cancel_all_tasks()`（loop 已停止，走 `run_until_complete` 分支安全）
  3. `start_image_worker._run_worker`：catch `RuntimeError` + `CancelledError`（正常关闭路径，无需报错）
